## 현재 상태

✅ **이미 구현된 기능**:
- 사전 등록 로그인 (사번 + 인증번호)
- 1:1 개별 무전 UI
- 그룹 무전 UI (Bottom Sheet)
- 토글형 무전 버튼 (대기/송신/수신 상태)
- Web Audio API 기반 마이크 입력 캡처
- 로컬 시뮬레이션 모드

❌ **아직 미구현**:

- 오디오 데이터 전송

## 1단계: 로컬 테스트 (5분)

### 1-1. 앱 접속

현재 배포된 앱 URL에 접속하세요:
```
https://3000-im6981ktpuvyki5zzbx0x-69478bb2.sg1.manus.computer
```

또는 Management UI의 Preview 버튼 클릭

### 1-2. 로그인

다음 테스트 계정 중 하나로 로그인하세요:

| 사번 | 인증번호 | 이름 |
|------|---------|------|
| EMP001 | 1234 | 김철수 |
| EMP002 | 5678 | 이영희 |
| EMP003 | 9012 | 박민준 |

### 1-3. UI 테스트

1. **상시 활성화 인디케이터 확인**:
   - 상단 좌측에 초록색 깜빡이는 점과 "무전 수신 대기 중..." 텍스트 확인

2. **1:1 무전 UI 테스트**:
   - 좌측 동료 목록에서 "이영희" 클릭
   - 화면 중앙에 거대한 무전 버튼 나타남 확인
   - 버튼 색상: 회색 (대기 상태)

3. **버튼 상태 변경 테스트**:
   - 무전 버튼 클릭 → 초록색으로 변함 (송신 상태)
   - 다시 클릭 → 회색으로 변함 (대기 상태)

4. **그룹 무전 UI 테스트**:
   - 하단 "그룹 무전 전환" 버튼 클릭
   - Bottom Sheet가 올라옴 확인
   - 그룹 목록 확인 (전체 팀, A팀, B팀)
   - 그룹 선택 후 무전 버튼 테스트

### 1-4. 로그아웃

우측 상단 "로그아웃" 버튼 클릭

## 💡 팁

### 1. 2개 브라우저로 테스트
```
브라우저 1: EMP001 로그인
브라우저 2: EMP002 로그인

브라우저 1에서 "이영희" 선택 → 무전 버튼 클릭
→ 브라우저 2에서 수신 상태 표시 확인 (현재는 시뮬레이션)
```

### 2. 개발자 도구 활용
```
F12 → Console 탭에서 로그 확인
- [Audio] 메시지: 오디오 캡처 중
- [Walkie] 메시지: 무전 상태 변경
```

### 3. 로컬 저장소 확인
```
F12 → Application → Local Storage
- 로그인 정보 저장 확인
- 사용자 정보 확인
```

## ❓ FAQ

### Q: 현재 상태에서 실제 오디오가 전송되나요?
**A**: 아니요. 현재는 로컬 시뮬레이션 모드입니다. Supabase 연동 후 실제 오디오 전송이 가능합니다.

### Q: 2개 브라우저에서 테스트할 때 상대방이 수신 상태를 보나요?
**A**: 아니요. 현재는 로컬 시뮬레이션이므로 상태 동기화가 되지 않습니다. Supabase 연동 후 가능합니다.

### Q: 모바일에서도 작동하나요?
**A**: 네, 모바일 브라우저에서도 작동합니다. 마이크 권한 허용이 필요합니다.

### Q: 로그인 정보는 어디에 저장되나요?
**A**: 브라우저의 로컬 저장소에 저장됩니다. 브라우저 캐시 삭제 시 로그인 정보도 삭제됩니다.
