# 그룹 무전 우선순위 기능 가이드

## 📌 개요

무전기 앱의 **그룹 무전 우선순위 기능**은 개별 무전 중에 그룹 무전이 수신되면 자동으로 개별 무전을 중단하고 그룹 무전을 우선 처리하는 기능입니다. 이는 긴급 상황에서 중요한 그룹 공지사항이 놓치지 않도록 보장합니다.

---

## 🎯 기능 설명

### 1. **개별 무전 중 그룹 무전 수신**

사용자가 특정 동료와 개별 무전을 하고 있을 때, 그룹 무전이 수신되면:

1. **자동 중단**: 진행 중인 개별 무전(송신 또는 수신)이 자동으로 중단됩니다.
2. **알림 표시**: 화면 상단에 주황색 배지가 나타나며 그룹 이름과 함께 "그룹 무전 우선" 표시가 됩니다.
3. **토스트 알림**: "A팀 그룹 무전이 우선됩니다!" 메시지가 표시됩니다.
4. **다이얼로그 팝업**: 그룹 무전 우선순위 알림 다이얼로그가 팝업되어 사용자에게 선택지를 제공합니다.

### 2. **사용자 선택지**

그룹 무전 우선순위 알림 다이얼로그에서 사용자는 두 가지 선택을 할 수 있습니다:

#### **옵션 1: 그룹 무전 계속 (권장)**
- 그룹 무전 Bottom Sheet가 자동으로 열립니다.
- 그룹 무전을 계속 진행할 수 있습니다.
- 그룹 무전이 종료되면 이전 개별 무전으로 돌아갈 수 있습니다.

#### **옵션 2: 개별 무전으로 복귀**
- 그룹 무전 우선순위가 해제됩니다.
- 이전에 선택했던 동료와의 개별 무전으로 돌아갑니다.
- 개별 무전 버튼이 다시 활성화됩니다.

### 상단 배지 (주황색)
```
⚠️ A팀 그룹 무전 우선
```
- **위치**: 화면 상단, "무전 수신 대기 중..." 옆
- **표시**: 그룹 무전 우선순위가 활성화되었을 때만 표시
- **색상**: 주황색 배경 + 주황색 테두리

### 우측 카드 내 안내 박스
```
⚠️ A팀 그룹 무전이 우선됩니다.
개별 무전으로 돌아가려면 아래 버튼을 클릭하세요.
[개별 무전으로 복귀] 버튼
```
- **표시**: 그룹 무전 우선순위 활성화 중
- **색상**: 주황색 배경
- **기능**: 사용자가 개별 무전으로 복귀할 수 있는 버튼 제공

### 그룹 무전 우선순위 알림 다이얼로그
```
그룹 무전 우선순위 알림

A팀 그룹에서 무전이 왔습니다.
어떻게 하시겠습니까?

[그룹 무전 계속] [개별 무전으로 복귀]
```
- **타이밍**: 그룹 무전 수신 시 자동 팝업
- **버튼**: 두 가지 선택지 제공

## 💡 사용 시나리오

### 시나리오 1: 긴급 공지
1. 사용자 A가 동료 B와 개별 무전 중
2. 전체 팀에서 긴급 공지 그룹 무전 수신
3. 개별 무전 자동 중단
4. 사용자가 "그룹 무전 계속" 선택
5. 전체 팀 그룹 무전 진행

1. 사용자 A가 동료 B와 개별 무전 중
2. 부서 공지 그룹 무전 수신
3. 개별 무전 자동 중단
4. 사용자가 "개별 무전으로 복귀" 선택
5. 동료 B와의 개별 무전 재개
6. 나중에 그룹 무전 내용 확인 가능

## 🧪 테스트 방법

### 개발 중 그룹 무전 시뮬레이션

브라우저 콘솔에서 다음 명령을 실행하여 그룹 무전 수신을 시뮬레이션할 수 있습니다:

```javascript
// 콘솔에서 실행
__simulateGroupCall("A팀");
```

**예상 결과:**
1. 화면 상단에 주황색 배지 표시: "A팀 그룹 무전 우선"
2. 진행 중인 개별 무전 자동 중단
3. 토스트 알림: "A팀 그룹 무전이 우선됩니다!"
4. 그룹 무전 우선순위 알림 다이얼로그 팝업

## 🔧 기술 구현 상세

### 관련 파일

| 파일 | 역할 |
|------|------|
| `client/src/hooks/useGroupPriority.ts` | 그룹 무전 우선순위 상태 관리 훅 |
| `client/src/hooks/useSimpleWalkieAudio.ts` | 오디오 스트리밍 및 그룹 무전 수신 이벤트 |
| `client/src/components/GroupPriorityAlert.tsx` | 우선순위 알림 다이얼로그 컴포넌트 |
| `client/src/pages/WalkieApp.tsx` | 메인 앱 컴포넌트 (통합) |

### 상태 관리

```typescript
// useGroupPriority 훅에서 관리하는 상태
- isGroupCallActive: 그룹 무전 우선순위 활성화 여부
- activeGroupName: 현재 활성화된 그룹 이름
- previousIndividualUser: 이전 개별 무전 상대방 정보
```

### 이벤트 흐름

```
1. 그룹 무전 수신 감지
   ↓
2. useSimpleWalkieAudio.simulateGroupCallReceived() 호출
   ↓
3. useGroupPriority.receiveGroupCall() 호출
   ↓
4. 개별 무전 중단 (audio.stopTransmitting/stopReceiving)
   ↓
5. isGroupCallActive 상태 변경
   ↓
6. UI 업데이트 (배지, 다이얼로그 표시)
```

## ⚙️ Supabase Realtime 연동 (향후)

현재는 로컬 시뮬레이션을 사용하지만, Supabase Realtime 연동 시:

```typescript
// Supabase Realtime 채널 구독
const channel = supabase.channel('group-calls');

channel.on('broadcast', { event: 'group_call_received' }, (payload) => {
  const { groupName } = payload;
  audio.simulateGroupCallReceived(groupName);
}).subscribe();
```

## 📞 문제 해결

### Q: 그룹 무전 우선순위 알림이 표시되지 않습니다.
**A:** 브라우저 콘솔에서 `__simulateGroupCall("테스트그룹")`을 실행하여 테스트하세요.

### Q: 개별 무전으로 복귀 버튼이 작동하지 않습니다.
**A:** 브라우저 개발자 도구의 콘솔에서 에러 메시지를 확인하고, `useGroupPriority` 훅의 상태를 점검하세요.

### Q: 그룹 무전 중 다른 그룹 무전이 수신되면?
**A:** 현재 그룹 무전이 우선이므로 새로운 그룹 무전은 무시됩니다. 현재 그룹 무전 종료 후 새 그룹 무전을 수신할 수 있습니다.
